Xiaoh帮大家搓了一个mini的可视化工具，位置在`resources/visualizer.py`。考虑到Python本身安装和环境配置的复杂性，我也帮各位把运行脚本写好了，位置就在下发文件根目录的`compile_and_visualize.bash`。

## 关于环境要求

如果你希望运行Visualizer，请检查如下环境是否正常：

- 项目根目录下**没有**名称为`build`和`.venv`的目录。
- 使用wsl的终端运行（如果你的CLion默认终端不是wsl，可以在设置的Tools-Terminal里把cmd/powershell改成C:/Windows/system32下的`wsl.exe`。

- 你的`main()`函数里只有`VisualizeTest();`这一行代码，没有加奇怪的输出/调试信息，不然会报错/出奇奇怪怪的问题。
- 不要修改下发文件中各个文件的位置/名字和CMakeLists.txt，否则可能出奇奇怪怪的东西。

## 关于数据

数据在`testcases`目录下，可视化的数据在`testcases/visualize_tests`的目录下，有$10$组助教挑选出来的可视化效果比较不错的tests。

如果你们希望对程序的正确性做基本检查，`testcases/visualize_tests`下也有对应的`.ans`文件表示输出。但是考虑到输出不唯一，你可能需要手动检查你的输出和标准答案是否一致。

对A班/想挑战`pressure tests`的B班同学，压力测试数据在`testcases/pressure_tests`下。只需要把`main()`函数里面的函数由`visualize()`修改为`PressureTest()`即可测试。**需要注意的是压力测试只测试运行速度而不测试代码的正确性**。

当然，你也可以自己编写一些测试函数。

## 关于可视化脚本

你不需要手动运行`visualizer.py`，因为这可能导致某些库不存在或有其他的错误。如果你需要运行可视化，假如你希望运行`visualize_tests`下的`large1.in`，你可以使用如下命令（请在项目的根目录下运行这条命令）：

```bash
bash compile_and_visualize.bash testcases/visualize_tests/large1.in
```

脚本会自动帮你配置环境并运行。在安装`Python`等情况下，可能会需要你输入wsl用户密码，跟着输就行了。

如果你的程序跑的比较慢（直观上就是模拟器播放速度很慢，拖速度条也没用），或者还没写完但是想玩一玩，Xiaoh也贴心的帮你准备了一份std的可执行文件。你可以使用如下命令使用助教自己的代码带动`visualizer`（请在项目的根目录下运行这条命令）：

```bash
bash compile_and_visualize.bash large1.in --standard-visualizer
```

我们非常建议你尝试一下可视化所有`large`测试点和`turing_machine.in`。

## 关于可视化工具

可视化工具支持多种功能，包括：实时的地图缩放，拖动，暂停，$\frac{1}{32}$~$128$x的速度倍率切换，以及最大化。如果在运行时一直显示`Waiting for frames`，请检查一下wsl运行时有没有报错，以及自己的输入输出有没有出奇怪的问题。

实时缩放使用`鼠标滚轮`控制，暂停使用`space键`控制，速度倍率由下方的速度控制条进行控制。

一些feature:

- 由于Python阴间的运行速度，在跑较大数据（特指large测试点）时可能会有一些卡顿，这是由渲染游戏地图的速度瓶颈导致，是正常现象，请各位多担待QAQ

- 尽管Xiaoh努力地在调整游戏默认的缩放和聚焦，但是程序在中小数据下默认的视野可能看不到图样，如果你找不到游戏地图，你可以试试如下操作：
  - 对于`small`部分的数据和`middle1`，尝试缩小一些地图，应该可以大致区域。
  - 对于`middle2`和`middle3`，为了较好的展示效果，地图被手动拉长了。`middle2`的图样在右侧，可以尝试缩小后往左拖动，在右边寻找图案。`middle3`的图样在下侧，可以尝试缩小后向上拖动，在下面寻找图案。
  - 对于`large`部分数据，在助教本机测试时默认缩放基本没有问题。

- Visualizer默认的播放速度比较慢，对于部分地图来说快放效果会比较好。

## 关于Debugging

首先，你可以基于visualize_tests你程序的输出和标准输出比对检查自己的正确性。

如果不希望自己手动去对比数据，可以尝试在超慢倍速下利用visualizer来进行可视化的调错。利用`space`控制暂停和播放。

或者，你可以尝试自己写一部分测试代码来进行调错。
